Chapter 01

概要

HTP-DB は、人類が各時代に生み出した文字・テキストの総量を体系的に推定するデータベースである。楔形文字の発明（紀元前 3200 年頃）から、AI 生成テキストが人間生成テキストと並走しはじめた現代（2022 年以降）までの約 5,200 年を、媒体・言語圏・推定方法ごとに記録する。

Wave 1-4 完走により、5,200 年の全期間を被覆した。Wave 1（1450-2026）は印刷とデジタルを Buringh & van Zanden（2009）+ UNESCO + IDC で固める。Wave 2（500-1450）は Buringh（2011）の中世写本期推計を中核に、宋代印刷術と古典イスラム期で三極バランスを取る。Wave 3（BC3200-AD500）は CDLI 楔形文字・Oxyrhynchus パピルス・Perseus/TLG 古典コーパスで古代を埋める。Wave 4 はサンスクリット・マヤ・ゲエズと AI 生成テキスト（2023-）を追加し、欧米中心からの脱却を試みた。

このデータベースは、ミラツクが構想する補論「情報生産の 2030-2100」の歴史根拠層として機能する。動詞の転換、すなわち「書き写す → 印刷する → 発信する → 共に生成する」という四段階のシフトを、量的データで裏打ちすることを意図する。Wave 4 で AI 生成テキストが 2025 年に量的に人類のデジタルテキスト総量に並ぶ可能性を示唆する初期推定値を含めた。

補論

2025 交差点の補論 — 「共に生成する」が定着する境界年

本章は、HTP-DB Wave 4 に格納された「2025 年 AI 生成テキスト 1-20 京 tokens」という当初推定を、Epoch AI と OpenAI 公式統計を一次照合した結果として書き直したものである。結論を先に述べると、当初の「1-20 京」という数字は文脈なしには両義的であり、API 消費量と実生成量を区別すれば、それぞれ妥当な範囲が異なる。

研究検証の結果、2025 年における AI 生成テキストの量的全体像は、三つの層に分けて理解する必要がある。第一に、OpenAI / Anthropic / Google 等の API が処理した推論時 tokens 総量は、年間およそ 120-180 京（1.2×10¹⁷ - 1.8×10¹⁷）tokens に達する。これは OpenAI 単体の公式統計 1.5B tokens/min と OpenRouter の集約値から推算した値であり、信頼度は中程度である。第二に、AI が新規に生成し外部へ持続的に投稿された unique なテキスト量は、Epoch AI のデータ稀少性モデルからの推算で年間 500 兆-5,000 兆 tokens（5×10¹⁴ - 5×10¹⁵）とおおむね 100 倍小さい。第三に、人類が文字を発明してから 2024 年までに蓄積した累積テキスト総量は、Epoch AI が 300 兆 tokens（信頼区間 100-1,000 兆）と推計している。三つの層は数桁ずれており、「人類超過」を語るときに、どの層を比較しているかが命運を決める。

研究検証の要点

「1-20 京 tokens」は API 消費量と読めば妥当（中央値 1.5×10¹⁷）、実生成テキスト量と読めば過大（実際は 10¹⁵ 程度）。当初 HTP-DB seed は両者を混同していたため、Wave 4.5 で API 消費量と unique 生成量を別レコードに分離し、`htp_source_discrepancies` にギャップ比率 150 倍を明示した。これは path-1 真実層の原則（結論を 1 つに収束せず、層別固定で繋ぐ）の実装例である。

考察 — 動詞「共に生成する」の定着年

2025 年を境界年とする仮説には、限定的な意味で妥当性がある。新規に Web へ公開される記事のレベルでは、Graphite が 65,000 件のサンプルで AI 比率 52% を観測し、MIT/Oxford は 2026 年に 64% と予測している。フローのレベルでは確かに「共に生成する」が主流の動詞へ移行した。しかし累積ストックのレベルでは、AI が貢献している割合は依然 15% 程度であり、人類が 5,200 年かけて蓄積した 300 兆 tokens の質保証済みテキストとの差は、当面埋まらない。

この差をどう読むかが、ミラツクの事業設計を分ける。一つの読み方は、AI 生成記事が氾濫することで一次解釈・翻訳的編集の価値が逆に高まる、という方向である。連載「〜のかたち」（暮らし / 事業 / 経営 / いとなみ / 変化）はこの読み方に立脚しており、エッセ・センスの研究者対談はその実装例となる。もう一つの読み方は、AI 協働の生産性そのものを引き上げて、人類が 1 人で扱える研究量を 1,000 倍に拡張する、という方向である。codex-team / FVCP の並列リサーチ体制はこちらにあたる。HTP-DB はこの両方の読み方を量的に裏打ちする歴史根拠層として機能する。

動詞の境界年は、ある一日に一斉に切り替わるのではない。新規フロー、累積ストック、API 消費という三層のうち、新規フローが先に超え、累積ストックが最後に超える。各層の比率推移を継続観測することで、ミラツクは「翻訳的編集」と「協働的拡張」のどちらに重きを置くかを、データに基づいて再調整できる。本補論はその観測の入口に位置づけられる。
